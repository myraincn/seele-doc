Sharding in Seele¶

1. Introduction¶

To increase the capacity of Seele, we partition the Seele network into 4 shards. In each shard there is an independent blockchain. Transactions can be made across shards. The cross-shard transaction is also called debt.

The difference between regular transactions and debts is that regular transactions happen between two accounts on the same shard while debts happen between two accounts on different shards.

Under $GOPATH\src\github.com\seeleteam\go-seele\cmd\node\config\, we have 4 nodes: node1.json, node2.json, node3.json, node4.json. Each json file corresponds to the configuration of one shard. They are used to run Seele nodes. For details, see How to customize your node configurations.

2. How to send debt¶

  • You need an account with value to transfer (for example, 100000 fan) from shard1 account to shard2 account.
shard1 account: 0x4c10f2cd2159bb432094e3be7e17904c2b4aeb21
shard2 account: 0xd228e1dd02bdce9656b8e195a77e0da0b70daea1
